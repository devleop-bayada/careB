# 1. 비즈니스 모델 캔버스

## 1.1 고객 세그먼트 (Customer Segments)

### B2C — 보호자 (수요 측)

| 세그먼트 | 규모 추정 | 특성 |
|----------|-----------|------|
| **재가 방문요양 이용 가족** | 연 80만 가구+ | 65세 이상 부모 돌봄, 정기 방문요양 필요, 장기요양등급 보유 |
| **비급여 간병 수요 가족** | 연 30만 가구+ | 병원 입원 보호자, 비급여 간병인 자비 부담 |
| **맞벌이 부모간병 세대** | 연 50만 명+ | 40~55세 직장인, 부모 돌봄과 직장 병행 어려움 |
| **긴급/단기 돌봄 수요** | 월 5만 건+ | 퇴원 직후, 보호자 부재, 급성기 회복기 케어 |

### B2C — 요양보호사 (공급 측)

| 세그먼트 | 규모 추정 | 특성 |
|----------|-----------|------|
| **자격증 보유 미취업** | 약 50만 명 | 자격증 취득 후 기관 취직 대기, 부업 희망 |
| **기관 소속 투잡 희망** | 약 30만 명 | 기관 외 개인 매칭 수익 병행 원함 |
| **프리랜서 경력자** | 약 10만 명 | 기관 독립 후 직접 계약 선호 |

> **핵심 인사이트**: 요양보호사 자격증 누적 발급 수 약 200만 명, 실제 활동 인력 약 65만 명. 활동률 32%. 비활동 인력 풀이 공급 확보의 핵심 레버.

### 보호자에게

# 2. 수익 모델 상세 설계

## 2.1 B2C 수익 모델

### 2.1.1 기본 매칭 수수료

**설계 근거**: 맘시터 0~5% → 규모 확보 후 수익화 전략을 요양보호사 시장에 적용. 초기 낮은 수수료로 네트워크 효과 확보, Phase 2부터 단계적 인상.

| 구분 | 수수료율 | 적용 시기 | 산출 근거 |
|------|----------|-----------|-----------|
| Phase 1 (0~6개월) | 0% | 초기 사용자 확보 | 무료 매칭으로 공급·수요 동시 획득 |
| Phase 2 (6~18개월) | 3% | 기본 수수료 도입 | 맘시터 최저 수수료 벤치마크 |
| Phase 3 (18개월+) | 5~8% | 수수료 정상화 | 플랫폼 가치 입증 후 인상 |

**수수료 계산 예시**:
- 방문요양 1회 비용: 18,000원 (시간당, 장기요양급여 기준)
- 월 12회 이용 시 월 거래액: 216,000원
- 5% 수수료 기준: 월 10,800원 / 연 129,600원

**연간 매칭 수수료 목표 (Year 3)**:
- 월간 활성 매칭 건수: 50,000건
- 건당 평균 거래액: 18,000원
- 월 총 거래액: 9억 원
- 수수료율 5%: 월 4,500만 원 → 연 54억 원

---

### 2.1.2 구독 모델 (보호자 월정액 플랜)

Care.com 구독 모델($12.95~$38.95/월)을 한국 시장에 적용. 구독 고객은 매칭 수수료 면제 또는 할인 혜택 제공.

| 플랜 | 월 요금 | 연 요금 | 핵심 기능 |
|------|---------|---------|-----------|
| **베이직 (Basic)** | 무료 | 무료 | 요양보호사 프로필 열람, 1건/월 면접 제안 |
| **스탠다드 (Standard)** | 19,900원 | 178,800원 | 무제한 면접 제안, 매칭 수수료 3% (일반 5%), 우선 노출, 돌봄 일지 기록 |
| **프리미엄 (Premium)** | 39,900원 | 358,800원 | 수수료 0%, 전담 케어 코디네이터, 긴급 매칭 우선권, 가족 공유 앱, 건강 리포트 월 1회, 법률 상담 1회/년 |
| **프로케어 (ProCare)** | 89,900원 | 808,200원 | 전담 케어 매니저, 요양보호사 관리 대행, 월 1회 방문 점검, 의료 코디네이션 포함 |

**구독 수익 목표 (Year 3)**:
- 누적 가입자: 30,000명
- 플랜별 분포: 베이직 40%, 스탠다드 35%, 프리미엄 20%, 프로케어 5%
- 월 구독 수익: (30,000 × 0.35 × 19,900) + (30,000 × 0.20 × 39,900) + (30,000 × 0.05 × 89,900)

## 4.6 손익분기점 (BEP) 분석

### 월간 BEP 계산

| 고정비 항목 | 월 비용 |
|-------------|---------|
| 인건비 (20명, 연 12억 ÷ 12) | 1억 원 |
| 인프라/클라우드 | 1,250만 원 |
| 사무실 | 1,000만 원 |
| 기타 고정비 | 500만 원 |
| **월 총 고정비** | **1억 2,750만 원** |

**BEP 달성 조건 (수수료 5% 기준)**:
- 필요 월 거래액: 1억 2,750만 원 ÷ 5% = **25억 5,000만 원**
- 필요 월 활성 매칭 건수: 25억 5,000만 원 ÷ 18,000원 = **141,667건**
- 하지만 구독 수익 포함 시 BEP 달성 거래액 기준 대폭 하락.

**구독 수익 포함 BEP**:
- 구독 가입자 5,000명 (평균 구독료 30,000원) → 월 구독 수익 1.5억 원
- 고정비 1.275억 원 - 구독 수익 1.5억 원 = 순 고정비 -2,250만 원
- 즉, 구독 가입자 5,000명 달성 시 수수료 없이도 BEP 도달 가능.

**BEP 달성 시나리오별 일정**:

| 시나리오 | BEP 달성 시점 |
|----------|--------------|
| 보수적 (느린 성장) | Year 3 Q2 |
| 기본 (계획 대비) | Year 2 Q3 |
| 낙관적 (빠른 성장) | Year 2 Q1 |

| **급여 삭감** | 재정 악화로 급여 단가 인하 (-10%) | 매칭 거래액 감소 → 수수료 수익 감소 | 비급여 간병 비중 확대, 수수료 의존도 30% 이하 유지 |
| **자격 제도 강화** | 요양보호사 추가 자격 요건 강화 | 공급 감소, 등록 장벽 상승 | 교육 사업 강화 → 자격 취득 지원이 오히려 플랫폼 이점 |
| **플랫폼 규제 신설** | 돌봄 플랫폼 등록제/신고제 도입 | 초기 규제 비용 발생 | 사전 규제 기관 협력, 선제 등록·인증 취득 |
| **공공 확장** | 정부 무료 매칭 서비스 확대 | 저가 B2C 시장 잠식 | 프리미엄·B2B·B2G로 수익 구조 다각화 |

**모니터링 체계**:
- 보건복지부 법령 개정 모니터링 (월 1회 리포트)
- 국민건강보험공단 정책 협의체 참여
- 요양보호사 협회, 요양기관 단체와 정책 연대

## 8.2 가격 경쟁 리스크

| 리스크 | 시나리오 | 영향 | 대응 전략 |
|--------|----------|------|-----------|
| **후발 대형 플랫폼 진입** | 카카오, 네이버가 요양 매칭 서비스 출시 | 마케팅 경쟁 심화, CAC 급등 | 도메인 전문성·신뢰 선점, B2B 장기 계약으로 방어선 구축 |
| **수수료 덤핑** | 경쟁사 수수료 0% 전략 | 수수료 수익 감소 | 구독 수익 비중 높여 거래 기반 의존도 40% 이하로 설계 |
| **기관 직접 앱 개발** | 대형 요양기관이 자체 매칭 앱 출시 | B2B 고객 이탈 | SaaS 깊이 있는 기능 확보, 전환 비용(switching cost) 설계 |
| **요양보호사 처우 경쟁** | 경쟁사 수수료 0% + 보너스 지급 | 공급 이탈 | 커뮤니티·교육·스폰서드 등 비금전적 가치 강화 |

**수익 구조 방어 전략**:
- 거래 기반 수익 : 구독 기반 수익 = 40:60 (Year 3 목표)
- 구독 전환 시 2개월 무료로 초기 전환 장벽 낮추되 해지 장벽은 높게 설계
- 기관 SaaS는 1년 단위 계약, 데이터 이전 어렵게 설계 (Switching Cost)

## 8.3 탈플랫폼(직거래) 리스크

요양보호사와 보호자가 플랫폼을 우회하여 직접 거래하는 것은 맘시터 등 모든 돌봄 플랫폼의 핵심 리스크.

| 리스크 요소 | 발생 조건 | 대응 전략 |
|-------------|-----------|-----------|
| **직거래 유인** | 수수료가 높거나, 매칭 후 신뢰 관계 형성 시 | 수수료 5%로 최소화 + 직거래 시 보험·보증 미제공 명확히 안내 |
| **보증 부재** | 직거래 시 사고·분쟁 발생 | 플랫폼 내 매칭에만 배상책임보험 제공 (탈플랫폼 시 보호 없음) |
| **데이터 이전** | 연락처 교환 후 플랫폼 이탈 | 연락처 직접 노출 금지, 앱 내 메시징 시스템 강제화 |
| **장기 관계** | 3개월+ 동일 요양보호사 이용 시 | 장기 계약 할인 (3% 수수료) + 정기 이용 편의 기능으로 플랫폼 잔류 유인 |

**탈플랫폼 방어 메커니즘 설계**:
1. **보험 연동**: 플랫폼 매칭 건에만 1억 원 한도 배상책임보험 자동 적용
2. **돌봄 일지 기록**: 앱 내에서만 공식 기록 보관 → 분쟁 시 증거 자료
3. **가족 공유**: 돌봄 일지를 가족 전원과 공유 → 직거래 시 이 기능 소멸
4. **공단 청구 연동**: 장기요양 급여 청구를 플랫폼에서 대행 → 행정 편의로 이탈 방지
5. **감사 시스템**: 이상 패턴 감지 (매칭 후 앱 미사용 = 직거래 의심) → 경고 및 제재
