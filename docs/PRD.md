## 1. 제품 개요

### 1.1 제품 비전

> "가족의 노후를 안심하고 맡길 수 있는 대한민국 최고의 요양보호사 매칭 플랫폼"

바야다(BAYADA)는 요양보호사와 보호자/환자를 신뢰 기반으로 연결하는 양면 마켓플레이스이다. 단순 인력 소개를 넘어, 돌봄의 전 과정(매칭 → 계약 → 모니터링 → 평가)을 디지털화하고, 요양기관(B2B) 및 정부/지자체(B2G)까지 아우르는 시니어케어 생태계를 구축한다.

**플랫폼 구조:**
- 양면 마켓플레이스: 요양보호사(공급) ↔ 보호자/환자(수요)
- B2B: 요양기관(노인요양원, 주야간보호센터, 방문요양기관)
- B2G: 정부/지자체 공공 돌봄 사업 연계

### 1.2 목표 (Goals & Non-Goals)

**Goals (달성 목표):**
- G1: 출시 후 12개월 내 누적 요양보호사 회원 10만 명 확보
- G2: 출시 후 12개월 내 누적 보호자 회원 5만 명 확보
- G3: 매칭 성공률 85% 이상 달성
- G4: 보호자 만족도 NPS 50점 이상
- G5: 요양보호사 재이용률 70% 이상
- G6: v1.0 출시 후 6개월 내 매월 거래액 50억 원 달성
- G7: 장기요양보험 청구 연동으로 기관 업무 효율 30% 개선

**Non-Goals (범위 외):**
- NG1: 직접 고용 (플랫폼은 매칭만, 고용주는 보호자/기관)
- NG2: 의료 행위 중개 (의사 처방, 주사 등 의료 서비스 불포함)
- NG3: 해외 시장 v1.0~v2.0 범위 제외 (v3.0+ 검토)
- NG4: 요양원/요양병원 입소 중개 (방문요양 중심)
- NG5: 자체 물리치료사/작업치료사 매칭 (v2.0 이후 검토)

### 1.4 타겟 사용자

#### Primary User 1: 보호자 (가족)
- **연령**: 40~60대 (부모 돌봄 주체)
- **상황**: 부모님이 거동이 불편하거나 치매, 만성질환으로 돌봄이 필요한 자녀
- **니즈**: 신뢰할 수 있는 요양보호사를 빠르게 찾고, 돌봄 상태를 원격으로 확인하고 싶음
- **페인포인트**: 검증된 요양보호사 찾기 어려움, 돌봄 품질 불투명, 장기요양등급 신청 복잡

#### Primary User 2: 요양보호사
- **연령**: 30~60대 여성 (80%), 요양보호사 자격증 보유자
- **상황**: 방문요양 형태로 일하거나 기관 소속으로 활동 중
- **니즈**: 안정적인 일감 확보, 공정한 보수, 근무 일정 관리, 전문성 향상 교육
- **페인포인트**: 일감 불안정, 돌봄 기록 관리 번거로움, 경력 인정 어려움

#### Secondary User 1: 환자 (수혜자)
- **연령**: 65세 이상
- **상황**: 장기요양등급 1~5등급 또는 인지지원등급 보유자
- **니즈**: 편안하고 전문적인 돌봄, 자신의 상태를 가족에게 전달

#### Secondary User 2: 요양기관 담당자 (B2B)
- **역할**: 방문요양기관, 주야간보호센터 운영자 및 사회복지사
- **니즈**: 소속 요양보호사 스케줄 관리, 급여 정산, 급여청구 자동화

#### Secondary User 3: 지자체/공공기관 담당자 (B2G)
- **역할**: 시·군·구 노인복지 담당 공무원
- **니즈**: 지역 돌봄 인력 현황 파악, 공공돌봄 인력 매칭

## 2. 기능 명세 - 보호자 앱

### 2.1 회원가입 / 로그인

**기능명:** 보호자 회원가입 및 로그인

**설명:** 보호자가 소셜 로그인 또는 이메일로 가입하고, 휴대폰 본인인증을 통해 신원을 확인한다. 가입 완료 후 돌봄 니즈 등록 온보딩 플로우로 자동 진입한다.

**유저 스토리:**
- As a 보호자, I want to 카카오/네이버/애플 계정으로 빠르게 가입하고, so that 별도 계정 생성 없이 서비스를 이용할 수 있다.
- As a 보호자, I want to 휴대폰 본인인증을 완료하여, so that 플랫폼에서 내 신원이 보증됨을 요양보호사에게 보여줄 수 있다.

**수용 기준 (Acceptance Criteria):**
```
Given 미가입 사용자가 앱을 처음 실행했을 때
When 카카오 소셜 로그인 버튼을 탭하면
Then 카카오 OAuth 인증 후 BAYADA 계정이 생성되고 온보딩 화면으로 이동한다

Given 소셜 로그인 완료 후
When 본인인증 화면에서 휴대폰 번호를 입력하고 인증번호를 수신하면
Then 6자리 인증번호 입력 후 30초 이내에 인증이 완료되어야 한다

Given 이미 가입된 이메일/소셜 계정으로 재시도 시
When 중복 가입을 시도하면
Then "이미 가입된 계정입니다" 안내 및 로그인 유도 메시지를 표시한다

Given 로그인 상태에서
When 앱을 재실행하면
Then 자동 로그인되어 메인 화면으로 이동한다 (토큰 만료: 30일)
```

**우선순위:** P0
**예상 개발 기간:** 2주 (BE 1주 + FE 1주)

### 2.2 돌봄 니즈 등록

**기능명:** 돌봄 대상자 정보 및 돌봄 조건 등록

**설명:** 보호자가 돌봄이 필요한 환자(어르신)의 상태, 돌봄 유형, 필요 시간대, 특수 요구사항을 등록한다. 이 정보를 기반으로 AI 매칭 알고리즘이 적합한 요양보호사를 추천한다.

**유저 스토리:**
- As a 보호자, I want to 부모님의 건강 상태와 돌봄 조건을 상세히 입력하여, so that 우리 상황에 맞는 요양보호사를 추천받을 수 있다.
- As a 보호자, I want to 여러 개의 돌봄 프로필을 생성하여, so that 부모님이 두 분 계신 경우 각각 관리할 수 있다.

**입력 항목:**
```
[환자 기본 정보]
- 이름 (별칭 사용 가능)
- 성별, 나이
- 장기요양등급 (1~5등급 / 인지지원등급 / 미신청)
- 주요 질환 (다중 선택): 치매, 뇌졸중, 파킨슨, 당뇨, 고혈압, 골절/관절염, 암, 기타

[돌봄 유형]
- 방문요양 (일상생활 지원)
- 방문목욕
- 방문간호 (연계)
- 주야간보호 연계
- 복합 케어

[일상생활 수행 능력]
- 식사: 자립 / 부분도움 / 완전도움
- 이동: 자립 / 보조기구 필요 / 완전도움
- 화장실: 자립 / 부분도움 / 기저귀 착용
- 목욕: 자립 / 부분도움 / 완전도움
- 인지: 정상 / 경도인지장애 / 치매

[돌봄 조건]
- 희망 지역 (시/구/동 단위)
- 돌봄 요일 (복수 선택)
- 돌봄 시간대 (시작 시간 ~ 종료 시간)
- 주당 돌봄 시간
- 희망 시작일

[특수 요구사항]
- 경력 최소 요건 (없음 / 1년 이상 / 3년 이상 / 5년 이상)
- 성별 선호 (무관 / 여성 / 남성)
- 흡연 여부 (비흡연자 필수 여부)
- 반려동물 동거 여부
- 기타 요청사항 (자유 입력, 500자)

[보수]
- 희망 시급 또는 월급 (직접 입력 / 시세 확인)
- 4대보험 적용 여부
```

**수용 기준:**
```
Given 보호자가 돌봄 니즈 등록 화면에 진입했을 때
When 필수 항목(환자 기본 정보, 돌봄 유형, 지역, 시간)을 입력하고 저장하면
Then 돌봄 프로필이 생성되고 매칭 추천 화면으로 이동한다

Given 장기요양등급을 '미신청'으로 선택했을 때
When 등록을 완료하면
Then 등급 신청 가이드 팝업을 노출하고, 등급 신청 안내 페이지로 연결 링크를 제공한다

Given 돌봄 니즈 등록 완료 후
When 조건을 수정하면
Then 수정 즉시 매칭 추천 목록이 갱신된다
```

**우선순위:** P0
**예상 개발 기간:** 2주

### 2.3 요양보호사 검색 및 필터

**기능명:** 요양보호사 검색 및 다중 필터

**설명:** 보호자가 지역, 경력, 전문분야, 평점, 시급 등 다양한 조건으로 요양보호사를 검색하고 결과를 정렬한다.

**유저 스토리:**
- As a 보호자, I want to 우리 동네에서 치매 케어 경험이 있는 요양보호사를 필터링하여, so that 관련 경험자만 추려서 빠르게 비교할 수 있다.

**필터 항목:**
```
[기본 필터]
- 지역: 시/구/동 (반경 km 설정 가능)
- 가능 날짜/시간대
- 시급 범위 (슬라이더)

[전문성 필터]
- 전문 케어 분야 (다중 선택)
  치매케어 / 와상케어 / 재활보조 / 방문목욕 / 호스피스 / 정신건강
- 최소 경력 연수
- 자격증 보유 여부
  치매전문교육 이수 / 요양보호사 1급 / 사회복지사 / 간호조무사

[개인 특성 필터]
- 성별
- 연령대 (30대 / 40대 / 50대 / 60대)
- 흡연 여부
- 차량 보유 여부

[신뢰/평가 필터]
- 최소 평점 (4.0 이상 / 4.5 이상)
- 리뷰 수 (10개 이상 / 30개 이상)
- 인증 배지 보유

[정렬 옵션]
- 추천순 (AI 매칭 점수)
- 거리순
- 평점 높은 순
- 리뷰 많은 순
- 경력 많은 순
- 시급 낮은 순 / 높은 순
```

**수용 기준:**
```
Given 보호자가 검색 화면에서 필터를 적용했을 때
When 지역과 시간대 필터만 적용하면
Then 해당 조건에 맞는 요양보호사 목록을 카드형 UI로 표시하고, 결과 없을 시 "조건을 조정해보세요" 안내를 표시한다

Given 필터 적용 후 결과가 표시되었을 때
When 스크롤하면
Then 무한 스크롤로 추가 결과를 로드한다 (페이지당 20명)

Given 검색 결과에서 요양보호사 카드를 탭했을 때
When 프로필 상세 화면으로 이동하면
Then 이전 필터 상태가 유지되어 뒤로 가기 시 결과 목록이 그대로 표시된다
```

**우선순위:** P0
**예상 개발 기간:** 2주

### 2.4 요양보호사 프로필 상세

**기능명:** 요양보호사 상세 프로필 조회

**설명:** 요양보호사의 경력, 자격증, 소개 영상, 리뷰, 가능 시간표를 한눈에 확인하는 상세 페이지.

**표시 항목:**
```
[헤더]
- 프로필 사진 / 소개 영상 (최대 3분)
- 이름 (성+이름 이니셜 표시: 김○○)
- 평점 (별점) + 리뷰 수
- 활동 지역
- 즐겨찾기 버튼

[기본 정보]
- 경력 연수
```

### 2.5 매칭 요청 및 수락

**기능명:** 요양보호사에게 매칭 요청 발송

**설명:** 보호자가 원하는 요양보호사에게 돌봄 조건을 포함한 매칭 요청을 발송하고, 요양보호사가 수락/거절한다.

**유저 스토리:**
- As a 보호자, I want to 마음에 드는 요양보호사에게 바로 매칭 요청을 보내어, so that 빠르게 돌봄을 시작할 수 있다.
- As a 보호자, I want to 최대 5명에게 동시 요청을 보내어, so that 한 명이 거절해도 다른 요양보호사에게 기회가 있다.

**수용 기준:**
```
Given 보호자가 이용권을 보유한 상태에서 프로필 페이지에서 "매칭 요청" 버튼을 탭했을 때
When 돌봄 조건 확인 화면에서 요청 메시지를 작성하고 발송하면
Then 요양보호사에게 푸시 알림이 발송되고, 요청 상태가 "대기중"으로 표시된다

Given 매칭 요청이 발송된 후
When 요양보호사가 수락하면
Then 보호자에게 푸시 알림과 인앱 알림이 발송되고, 채팅방이 자동 생성된다

Given 매칭 요청이 발송된 후
When 48시간 이내에 요양보호사가 응답하지 않으면
Then 요청이 자동 만료되고 보호자에게 알림이 발송된다. 이용권은 반환된다.

Given 이용권 없이 매칭 요청을 시도하면
When 버튼을 탭하면
Then 이용권 종류와 가격을 안내하는 바텀시트가 노출된다
```

**우선순위:** P0
**예상 개발 기간:** 1주

### 2.6 상담/면접 예약 (화상/대면)

**설명:** 매칭 요청 수락 후 보호자와 요양보호사가 화상 또는 대면 면접을 예약하고 진행한다. 화상 면접은 인앱 영상통화로 지원한다.

**유저 스토리:**
- As a 보호자, I want to 실제 고용 전에 화상으로 요양보호사를 만나보고, so that 신뢰감과 적합성을 확인한 후 계약을 결정할 수 있다.

**수용 기준:**
```
Given 채팅방이 개설된 후
When 보호자가 "면접 예약" 버튼을 탭하면
Then 캘린더에서 가능 시간대를 선택하는 화면이 노출되고, 선택한 일정이 요양보호사에게 제안된다

Given 면접 일정이 확정된 후
When 면접 30분 전이 되면
Then 보호자와 요양보호사 모두에게 리마인더 푸시 알림이 발송된다

Given 화상 면접 예약 시
When 예약된 시간에 "입장" 버튼을 탭하면
Then Agora/WebRTC 기반 화상통화가 시작된다 (최대 60분)

Given 면접 완료 후
When 보호자가 면접 결과를 선택하면
Then 계약 진행 / 다른 요양보호사 탐색 / 추가 면접 예약 중 선택할 수 있다
```

**우선순위:** P0

### 2.7 전자 계약 체결

**유저 스토리:**
- As a 보호자, I want to 앱 안에서 바로 계약서에 서명하여, so that 별도 서류 작성 없이 법적 효력 있는 계약을 체결할 수 있다.

**계약서 포함 항목:**
```
1. 당사자 정보 (보호자, 요양보호사)
2. 돌봄 대상자 정보
3. 서비스 내용 (돌봄 유형, 요일, 시간)
4. 보수 (시급/월급, 지급일, 지급 방법)
5. 4대보험 적용 여부
6. 계약 기간 및 갱신 조건
7. 해지 조건 (사전 통보 기간)
8. 비밀유지 조항
9. 개인정보 처리 동의
10. 분쟁 해결 방법
```

**수용 기준:**
```
Given 면접 완료 후 계약 진행을 선택했을 때
When 계약서 화면에 진입하면
Then BAYADA 표준 계약서 템플릿이 돌봄 조건에 맞게 자동 채워진다

Given 계약서 확인 후
When 보호자가 전자서명을 완료하면
Then 요양보호사에게 서명 요청 알림이 발송된다

Given 양측 모두 서명 완료 시
When 계약이 체결되면
Then PDF 계약서가 생성되고 양측 이메일로 발송되며, 앱 내 '내 계약' 탭에 저장된다

Given 계약 체결 후
When 보호자가 계약 기간 중 수정을 원하면
Then 요양보호사 동의 후 계약 변경서를 추가 서명한다
```

### 2.8 실시간 돌봄 모니터링

**기능명:** 돌봄 일지 및 실시간 활동 보고

**설명:** 요양보호사가 돌봄 중 작성한 일지를 보호자가 실시간으로 확인한다. 출근/퇴근 GPS 인증, 활동 내역, 건강 상태를 모니터링한다.

**유저 스토리:**
- As a 보호자, I want to 직장에 있는 동안 요양보호사가 부모님께 무엇을 하고 있는지 실시간으로 확인하여, so that 안심하고 일에 집중할 수 있다.

**표시 정보:**
```
[오늘의 돌봄 현황]
- 출근 시간 (GPS 인증 시간)
- 퇴근 시간
- 총 돌봄 시간
- 현재 상태 (돌봄 중 / 종료)

- 활동: 산책, 운동, 여가 활동
- 건강 상태: 체온, 혈압, 혈당 (해당 시)
- 투약 관리: 약물명, 복용 시간, 복용 여부
- 배변/위생: 횟수, 특이사항
- 정서 상태: 기분, 수면, 특이 행동
- 특이사항: 낙상, 통증 호소 등 이상 증상

[사진/영상]
- 일지와 함께 첨부 사진 (보호자 동의 설정에 따라)
```

**수용 기준:**
```
Given 요양보호사가 돌봄 시작 GPS 인증을 완료했을 때
When 보호자 앱의 홈 화면을 확인하면
Then "돌봄 중" 상태 배지와 출근 시간이 표시된다

Given 요양보호사가 돌봄 일지를 작성하고 저장했을 때
When 보호자가 앱을 확인하면

When 보호자 앱에서 알림을 수신하면
Then 즉시 푸시 알림과 함께 "긴급 확인 필요" 배지가 표시된다
```

### 2.11 채팅 및 영상통화

**기능명:** 보호자-요양보호사 채팅 및 영상통화

**설명:** 매칭 수락 이후 보호자와 요양보호사가 채팅으로 소통하고, 필요 시 영상통화를 할 수 있다. 돌봄 중 환자 상태를 영상으로 확인하는 용도로 활용된다.

**기능 상세:**
```
[채팅]

- 음성 메시지 (최대 2분)

- 채팅방 별 알림 설정
- 메시지 번역 (한국어 기본)
```

**수용 기준:**
```
Given 매칭이 수락된 후
When 채팅방에 진입하면

Then 3초 이내에 영상이 연결된다 (Agora SDK 활용)

Given 채팅 메시지 발송 시
When 상대방이 앱을 사용 중이 아닐 때
Then 푸시 알림으로 메시지 내용 일부가 전달된다 (민감 정보 미포함)
```

### 2.12 긴급 돌봄 요청

**기능명:** 즉시 매칭 긴급 돌봄 서비스

**설명:** 예약 없이 당일 또는 익일 긴급하게 돌봄이 필요한 경우, 가용한 요양보호사에게 긴급 요청을 발송한다.

**유저 스토리:**
- As a 보호자, I want to 갑자기 부모님 돌봄이 필요한 상황에서 빠르게 요양보호사를 구하여, so that 응급 상황에 대응할 수 있다.

**수용 기준:**
```
Given 프리미엄/패밀리케어 구독자가 긴급 돌봄 버튼을 탭했을 때
When 긴급 돌봄 조건(지역, 시간, 시간수)을 입력하고 요청하면
Then 해당 지역의 가용 요양보호사 5명에게 동시에 긴급 요청 알림이 발송된다

Given 긴급 요청 후
When 요양보호사 중 1명이 수락하면
Then 보호자에게 즉시 알림이 발송되고 채팅방이 열린다

Given 30분 내 수락자가 없을 경우
When 자동 만료되면
Then 보호자에게 "매칭 실패" 알림과 함께 케어 매니저 연결 옵션이 제공된다
```

**우선순위:** P1 (프리미엄 구독자 전용)
**예상 개발 기간:** 1주

## 3. 기능 명세 - 요양보호사 앱

### 3.1 회원가입 및 자격 인증

**기능명:** 요양보호사 자격 인증 및 가입

**설명:** 요양보호사가 자격증, 경력, 건강검진 서류를 업로드하고 BAYADA 운영팀이 인증하는 프로세스.

**유저 스토리:**
- As a 요양보호사, I want to 내 자격증과 경력을 플랫폼에서 인증받아, so that 보호자에게 신뢰를 줄 수 있다.

**인증 서류 항목:**
```
[필수]
- 요양보호사 자격증 (정부24 또는 사본)
- 신분증 (주민등록증 또는 운전면허증)
- 본인 명의 계좌 (정산용)
- 성범죄·아동학대 경력 조회 동의서

[선택 (배지 부여)]
- 건강검진 결과서 (최근 1년 이내)
- 치매전문교육 이수증
- 경력증명서 (이전 기관 발급)
- 추가 자격증 (사회복지사, 간호조무사 등)
```

**인증 프로세스:**
```
Step 1: 기본 정보 입력 (이름, 생년월일, 연락처)
Step 2: 본인인증 (휴대폰 인증)
Step 3: 서류 업로드 (사진 촬영 또는 파일 선택)
Step 4: 성범죄 조회 동의 (전자 동의)
Step 5: 운영팀 검토 (영업일 기준 1~2일)
Step 6: 인증 완료 및 프로필 활성화
```

**수용 기준:**
```
Given 요양보호사가 서류를 업로드했을 때
When 운영팀이 인증을 완료하면
Then 요양보호사에게 푸시 알림이 발송되고 프로필이 검색 결과에 노출된다

Given 서류 검토 결과 반려 시
When 반려 사유와 함께 재업로드 요청이 발송되면
Then 요양보호사가 수정 후 재제출할 수 있다

Given 요양보호사 자격증 유효성 확인 시
When 자격번호를 입력하면
Then 사회보장정보시스템 API 연동으로 자격증 유효성이 자동 확인된다 (가능 시)
```

**우선순위:** P0
**예상 개발 기간:** 2주

```
[기본 정보]
- 프로필 사진

- 이동 수단 (도보 / 대중교통 / 자차)

- 휴가/비가용 날짜 설정 (캘린더)

[보수 설정]
- 희망 시급 / 월급 설정
- 4대보험 적용 여부
```

```
Given 소개 영상을 업로드했을 때
When 업로드가 완료되면
Then 영상이 자동으로 가로 최적화되고 섬네일이 자동 생성된다

Given 가용 시간을 변경했을 때
When 저장하면
Then 보호자 검색 결과에 즉시 반영된다
```

**우선순위:** P0

**설명:** 보호자의 매칭 요청을 수신하고 수락 또는 거절한다. 거절 시 사유를 선택하여 플랫폼 데이터로 활용한다.

**거절 사유 옵션:**
```
- 해당 기간 스케줄 불가
- 이동 거리가 너무 멈
- 케어 조건이 제 전문분야와 맞지 않음
- 다른 매칭을 우선 진행 중
- 기타 (직접 입력)
```

**수용 기준:**
```
Given 매칭 요청을 수신했을 때
When 푸시 알림을 탭하면
Then 보호자의 돌봄 조건 요약 화면이 표시된다 (환자 상태, 지역, 시간, 시급)

Given 수락 버튼을 탭했을 때
When 수락이 완료되면
Then 채팅방이 생성되고 보호자에게 수락 알림이 발송된다

Given 매칭 요청을 48시간 이내에 응답하지 않으면
When 자동 만료되면
Then "응답 없음"으로 처리되고, 반복 시 검색 노출 순위가 낮아진다는 안내가 표시된다
```

**우선순위:** P0
**예상 개발 기간:** 1주

**일지 항목:**

```
[활동 기록]

- 신체 활동: 산책, 스트레칭, 재활 운동 여부
- 위생: 세면, 구강, 피부 관리
- 배변: 횟수, 형태, 특이사항
- 목욕: 해당 일만 표시

[건강 상태]
- 체온 (℃)
- 혈압 (mmHg) - 수축기/이완기
- 혈당 (mg/dL) - 해당 시
- 산소포화도 (%) - 해당 시
- 통증 호소 여부

[투약 관리]
- 복용 약물 목록 (사전 등록)
- 복용 여부 (복용 / 미복용)
- 미복용 사유

[정서 및 인지]
- 전반적 기분 (좋음 / 보통 / 우울)
- 인지 상태 (맑음 / 혼란 / 무기력)
- 수면 상태 (어제 밤 기준)

[특이사항]
- 자유 입력 (낙상, 욕창, 이상 행동, 통증 증가 등)
- 사진 첨부 (최대 5장)

[사인 (GPS 인증)]
- 돌봄 시작 GPS 인증
- 돌봄 종료 GPS 인증
```

**수용 기준:**
```
Given 요양보호사가 돌봄 종료 GPS 인증을 했을 때
When 일지 작성 화면이 자동으로 열리면
Then 당일 돌봄 시작 시간이 자동 입력되어 있다

Given 일지를 제출했을 때
When 제출이 완료되면
Then 보호자에게 푸시 알림이 발송되고, 일지 내용이 보호자 앱에 표시된다

Given 특이사항(낙상, 이상 증상)이 기록된 경우

Then 보호자에게 즉시 긴급 알림이 발송된다 (일반 알림과 구분)
```

```
    "meta": { "page": 1, "total": 100 }
```

## 9. 부록

### 9.1 용어 정의

| 용어 | 정의 |
|------|------|
| 요양보호사 | 국가 자격증 보유자로 노인 및 장애인의 일상생활을 지원하는 전문 돌봄 인력 |
| 장기요양등급 | 국민건강보험공단에서 판정하는 1~5등급 및 인지지원등급. 등급에 따라 요양급여 수급 |
| 장기요양급여 | 장기요양등급 판정자가 이용 가능한 재가급여, 시설급여, 특별현금급여 |
| 재가급여 | 자택에서 이용하는 방문요양, 방문목욕, 방문간호, 주야간보호 등의 서비스 |
| 방문요양 | 요양보호사가 수급자 자택을 방문하여 신체 활동 및 가사 활동을 지원하는 서비스 |
| 수급자 | 장기요양등급 인정을 받아 장기요양급여를 받는 사람 (환자) |
| 보호자 | 수급자의 가족으로 돌봄 서비스를 요청하고 계약하는 사람 |
| 매칭 | 보호자의 돌봄 니즈와 요양보호사를 연결하는 과정 |
| 돌봄 일지 | 요양보호사가 매 돌봄 후 작성하는 활동 기록 |
| GPS 인증 | 요양보호사의 돌봄 시작/종료 시 위치를 확인하는 절차 |
| 이용권 | 보호자가 요양보호사에게 매칭 요청 1회를 발송할 수 있는 권리 |
| NPS | Net Promoter Score. 고객 추천 의향 지수 |
| MAU | Monthly Active Users. 월간 활성 사용자 수 |
| LTV | Lifetime Value. 고객 생애 가치 |
| CAC | Customer Acquisition Cost. 고객 획득 비용 |
