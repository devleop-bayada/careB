## 2. 유저 플로우 기반 개발 순서

### Phase 1: 핵심 유저 플로우 완성 (P0 Gap 해소)

> 목표: 보호자가 가입 → 검색 → 매칭 → 상담 → 계약까지 끊김 없이 진행 가능

#### Step 1-1: 매칭 요청 페이지 생성 🔴
- **명세서**: FS-G-005
- **작업**: `src/app/(app)/matching/new/page.tsx` 생성
  - 요양보호사 선택 정보 표시
  - 돌봄 조건 확인/수정 폼
  - 요청 메시지 작성
  - 매칭 요청 발송 (POST /api/matches)
- **크기**: M
- **의존성**: 없음 (최우선)

#### Step 1-2: 프로필 수정 저장 버튼 추가 🔴
- **명세서**: FS-C-002
- **작업**: `src/app/(app)/my/profile/page.tsx`에 submit 버튼 + 저장 로직
- **크기**: S
- **의존성**: 없음

#### Step 1-3: 계약서 API + 서명 플로우 🔴
- **명세서**: FS-G-007
- **작업**:
  - `src/app/api/contracts/[id]/sign/route.ts` 생성 (또는 기존 contracts API 수정)
  - contract 페이지에서 올바른 API 호출
  - 양측 서명 상태 관리
- **크기**: M
- **의존성**: Step 1-1 (매칭 완료 후 계약 진행)

#### Step 1-4: 홈 AI 추천 실데이터 연동
- **명세서**: FS-G-003
- **작업**: `src/app/(app)/home/page.tsx` — 하드코딩 `[95, 91, 87]` → `/api/caregivers/recommend` API 호출
- **크기**: S
- **의존성**: 없음

#### Step 1-5: 알림 클릭 이동 + 읽음 처리
- **명세서**: 공통
- **작업**:
  - `src/app/(app)/notifications/page.tsx` — 알림 클릭 시 관련 페이지 이동
  - PATCH `/api/notifications/[id]` — 읽음 처리 API
- **크기**: S
- **의존성**: 없음

#### Step 1-6: 마이페이지 Dead 링크 수정
- **작업**: `src/app/(app)/my/page.tsx`
  - `href="#"` → 실제 페이지 링크 또는 바텀시트
  - 약관, 개인정보처리방침: 정적 페이지 생성
  - 고객센터: 카카오톡 채널 또는 이메일 링크
  - 회원 탈퇴: 탈퇴 확인 다이얼로그 + API 연동
- **크기**: M
- **의존성**: 없음

#### Step 1-7: 소셜 로그인 연동 (카카오/네이버)
- **명세서**: FS-G-001
- **작업**:
  - NextAuth에 Kakao, Naver provider 추가
  - OAuth 콜백 처리
  - `src/app/(auth)/login/page.tsx` — onClick 핸들러 연결
- **크기**: M
- **의존성**: 없음

#### Step 1-8: 결제/구독 기본 연동
- **명세서**: FS-G-008
- **작업**:
  - Prisma: Payment, Subscription, Pass 모델 추가
  - 토스페이먼츠 SDK 연동
  - 이용권 구매 플로우 구현
  - 구독 플랜 선택/결제 화면
- **크기**: L
- **의존성**: 없음 (but 매칭 요청 시 이용권 검증과 연결)
